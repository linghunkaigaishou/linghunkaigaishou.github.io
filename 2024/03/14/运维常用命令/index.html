windows常用命令

端口转发

1.设置端口转发规则：

增加一条端口转发规则：

netsh interface portproxy add v4tov4 listenaddress=本地IP listenport=本地侦听端口 connectaddress=远程IP connectport=远程端口

删掉一条端口转发规则：

netsh interface portproxy delete v4tov4

清空当前所有端口转发配置规则：

netsh interface portproxy reset

2.查询当前所有端口转发配置规则：

netsh interface portproxy show all

FAQ:

  1. 若端口转发配置在系统重启以后重置，考虑在计划任务中添加开机配置端口转发任务。
  2. 端口转发依赖的ip helper服务是否启动
  3. 防火墙规则检查

Linux常用命令

创建sudo用户

1.创建新用户:

useradd 新用户

2.设置该用户密码：

passwd 新用户

3.将新用户添加到sudo组，以获得管理员特权：

usermod -aG wheel 新用户

FAQ：

默认情况下，在 CentOS 系统上，组成员wheel被授予 sudo 访问权限

端口转发（基于firewall）

1.查看firewalld运行状态：

systemctl status firewalld

2.添加一条端口转发规则：

firewall-cmd –add-forward-port=port=本地侦听端口:proto=tcp:toport=远程端口:toaddr=远程IP –permanent

移除一条端口转发规则：

firewall-cmd –remove-forward-port=port=本地侦听端口:proto=tcp:toport=远程端口:toaddr=远程IP –permanent

3.允许信任IP访问侦听端口：

firewall-cmd –permanent –zone=public –add-rich-rule=”rule family=”ipv4” source address=”信任IP地址段” port protocol=”tcp” port=”侦听端口” accept”

或者

firewall-cmd –zone=public –add-port=侦听端口/tcp –permanent

4.开/关地址伪装功能（NAT）：

开启地址转发功能：

firewall-cmd –add-masquerade –permanent

查看包转发功能是否开启（默认为0-未开启，1-包转发开启）：

sysctl net.ipv4.ip_forward

关闭地址转发功能：

firewall-cmd –del-masquerade –permanent

或者

firewall-cmd –remove-masquerade –permanent

5.重新加载配置文件：

firewall-cmd –reload

6.查看防火墙规则列表：

firewall-cmd –list-all

FAQ:

默认没有开启包转发功能。即系统只处理目的地为本机的数据包，不会转发发往其它地址的数据包。在内核中有相应配置选项net.ipv4.ip_forward 默认设置为0。可以通过命令sysctl net.ipv4.ip_forward=1手动开启包转发功能。防火墙的masquerade功能进行地址伪装(NAT)，私网访问公网或公网访问私网都需要开启此功能来进行地址转换，否则无法正常互访。通过命令firewall-cmd –add-masquerade开启此功能，开启之后net.ipv4.ip forward的值自动被设置为1，即centos启用包转发。当然，如果启用了masquerade而不启用包转发的设置是没有意义的，相当于只有本机的数据能进出网络接口，那么NAT就没有意义了。

数据库常用命令

更新某条数据

update 表名 set 需要更新的列名 = ‘更新的列值’ where 过滤字段 = ‘过滤字段值’

举例：

update 表名 set ICKKH=’22719673797880’ where RYBH = ‘20220718’

统计数据条数

SELECT COUNT(*) FROM 数据表名;
